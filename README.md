## 💡 Executive Summary
The Recommendation Serving & Analytics Platform solves the **"latency vs. scale"** trade-off. Traditional methods of querying large datasets (Athena/SQL) often result in multi-second latencies.

This platform utilizes an **In-Memory Cache-Aside** architecture. The serving layer maintains a pre-processed mirror of the S3 data source in RAM, enabling **sub-10ms response times**. 

**Key Recent Enhancements:**
- **AI-Powered ChatBot**: Natural language interface for querying dataset insights and specific recommendations.
- **Robust RBAC**: Role-Based Access Control for secure user and permission management.
- **Feedback Loop**: Integrated system for recording real-time recommendation feedback to improve model calibration.

## 🏁 Getting Started

### 1. Prerequisites
- Python 3.12+
- Node.js 18+
- MongoDB (Local or Atlas)
- AWS Credentials (configured via `.env` or IAM role)

### 2. Backend Setup
```bash
# Navigate to backend
cd backend

# Create virtual environment
python -m venv venv

# Activate venv (Windows)
.\venv\Scripts\Activate.ps1

# Install dependencies
pip install -r requirements.txt

# Configure Environment
cp .env.example .env  # Update with your AWS/Mongo keys

# Start the backend Server
uvicorn main:app --reload
```

### 3. Frontend Setup
```bash
# Navigate to frontend
cd frontend

# Install dependencies
npm install

# Start the development server
npm run dev
```

### 4. Running the Project
- **Backend**: `uvicorn main:app --reload --workers 1`
- **Frontend**: Access via `http://localhost:5173`
- **Default Login**: `admin` / `admin123`

## 🏗 System Architecture

### High-Level Design
The system is split into three distinct tiers:
1.  **Storage Tier (AWS S3)**: The immutable source of truth. Data is stored as Parquet/CSV files generated by SageMaker.
2.  **Serving Tier (FastAPI)**: Loads the entire recommendation set into optimized memory structures.
3.  **Presentation Tier (React)**: Modular dashboard with high-frequency polling and AI chat interface.

### The "Load-to-Memory" Strategy
Instead of querying a database per request, the backend performs a **Bulk Load** during startup:
- **Time Complexity**: $O(N)$ during startup load.
- **Lookup Complexity**: $O(1)$ constant time for customer retrieval via hash mapping.

## 🔍 Deep Dive Technical Details

### Data Loading Engine (`data_loader.py`)
Loaded data is stored in a `defaultdict(list)` structure, allowing instantaneous retrieval by `customer_id`.

### RBAC & Security
The system implements a granular permission system. Each user is assigned a **Role**, and each Role is mapped to a set of **Permissions** (e.g., `user_manage`, `data_refresh`, `stats_view`). Access is enforced via FastAPI dependencies (`PermissionChecker`).

### AI ChatBot (`chatBot/`)
An LLM-powered assistant that translates natural language questions into Python queries against the in-memory DataFrame. It uses a two-step process:
1. **Intent Detection**: Classifies the query (e.g., `customer_recommendation`, `summary`).
2. **Truthful Execution**: For data-sensitive queries, it returns raw Python results to prevent hallucinations. For summaries, it uses the LLM to format the data professionally.

## 📊 Model Governance

- **Quality Mix**: Tracks the balance between **Association-based** (High Signal) and **Fallback** (Generic) recommendations.
- **Model Health**: Tracks **Silhouette Scores** to monitor the clustering quality of the underlying model.
- **Feedback Integration**: Real-time user feedback is flushed to S3 to allow the SageMaker pipeline to calibrate the next model iteration.

| `CORS_ORIGINS` | CORS origins | `http://localhost:5173` |

### ⚠️ Critical Warning: Worker Count
This platform uses an **in-memory cache**. If deploying in production, you **MUST** use a single worker (`--workers 1`). Multi-worker deployments require a distributed cache like Redis to maintain consistency during `/data/refresh` calls.
